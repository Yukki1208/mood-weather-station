# 匿名即时问答工具 (Anonymous Q&A Tool)

## 1. 项目概述

**项目名称**: 匿名问答 (AnonQA)
**项目类型**: 实时 Web 应用
**核心功能**: 面向课堂、讲座、社团活动的匿名即时问答系统。主持人扫码创建房间，参与者输入6位房间号匿名发言，消息以弹幕/卡片流形式实时展示。
**目标用户**: 教师、讲座主持人、活动组织者，以及学生、听众等参与者

## 2. 技术架构

### 前后端架构
- **后端**: Node.js + Express + Socket.IO
- **前端**: 原生 HTML/CSS/JavaScript + Socket.IO Client
- **通信**: WebSocket 实时双向通信

### 数据模型
```
Room {
  id: string (6位数字验证码)
  createdAt: timestamp
  messages: Message[]
  isActive: boolean
}

Message {
  id: string
  content: string
  timestamp: timestamp
  isPinned: boolean
}
```

## 3. 功能列表

### 主持人功能
1. **创建房间** - 生成6位唯一房间号
2. **查看问答列表** - 实时显示所有匿名消息
3. **置顶消息** - 将重要消息置顶显示
4. **删除消息** - 移除不当内容
5. **结束问答** - 关闭房间

### 参与者功能
1. **加入房间** - 输入6位房间号加入
2. **发送匿名消息** - 无需登录即可发送
3. **实时接收** - 消息以弹幕/卡片形式实时展示

## 4. UI/UX 设计方向

### 整体风格
- 简约现代，采用卡片式设计
- 暗色主题，适合投影/大屏展示

### 颜色方案
- 主色: #6366f1 (靛蓝色)
- 次色: #8b5cf6 (紫罗兰)
- 背景: #0f172a (深蓝黑)
- 卡片背景: #1e293b
- 文字: #f8fafc
- 强调色: #22c55e (在线/成功)
- 警告色: #ef4444 (删除/错误)

### 弹幕/卡片流设计
- 新消息从右侧滑入
- 消息卡片随机颜色边框
- 支持置顶消息固定显示
- 平滑的进入/退出动画

### 布局
- **主持人页面**: 左侧房间信息+控制面板，右侧消息流
- **参与者页面**: 上方输入框，下方消息流
- **展示模式**: 全屏卡片流，适合投影

## 5. 页面结构

### 入口页面 (index.html)
- 两个入口按钮：创建房间 / 加入房间

### 主持人页面 (host.html)
- 房间号展示（带二维码）
- 消息列表（可管理）
- 控制面板

### 参与者页面 (participant.html)
- 房间号输入
- 昵称/匿名设置
- 消息输入框
- 消息流展示

### 展示页面 (display.html)
- 全屏消息卡片流
- 适合投影/大屏展示

## 6. API 接口

### WebSocket 事件

**客户端 -> 服务端**
- `create-room` - 创建新房间
- `join-room` - 加入房间
- `send-message` - 发送消息
- `pin-message` - 置顶消息
- `delete-message` - 删除消息
- `close-room` - 关闭房间

**服务端 -> 客户端**
- `room-created` - 房间创建成功
- `room-joined` - 加入房间成功
- `new-message` - 新消息
- `message-pinned` - 消息被置顶
- `message-deleted` - 消息被删除
- `room-closed` - 房间已关闭
- `room-list` - 房间消息列表

## 7. 验收标准

1. ✅ 主持人可以创建房间并获得6位房间号
2. ✅ 参与者可以通过房间号加入房间
3. ✅ 匿名消息可以实时发送到所有人
4. ✅ 消息以卡片流/弹幕形式展示
5. ✅ 主持人可以置顶/删除消息
6. ✅ 页面适配PC和移动端
7. ✅ 界面美观，适合投影展示
